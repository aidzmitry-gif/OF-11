# OF-11 · Модуль «Офис-менеджер» (office)

Постпродажный документальный конвейер CMR-ERP: ведёт сделку после победы в CRM
по стадиям **отгрузка → сбор документов → оплата** в виде канбан-воронки.
В отличие от исходной заглушки, этот модуль **связан с пятью отделами** через
событийную шину и умеет **создавать заявку перевозчику на доставку по РБ** прямо
из карточки документа.

- **API-префикс:** `/office`  ·  **схема БД:** `office`  ·  **версия:** `0.2.0`
- **Стек:** FastAPI · SQLAlchemy (async) · Pydantic · событийная шина `core` (outbox)
- **Превью UI:** `office-manager.html` — самодостаточный HTML-макет (design-system CMR-ERP)

## Карта связей (офис в центре контура)

**Входящие** (подписки, см. `module.register`):

| Событие | Источник | Реакция офиса |
|---|---|---|
| `sales.deal.won` | CRM/Sales | Завести документ в стадии «Готово к отгрузке», сразу попросить Склад собрать |
| `wms.shipment.completed` | Склад | Двигать документ в «Отгружено» |
| `logistics.delivery.delivered` | Логистика | Зафиксировать трекинг/доставку в документе |
| `finance.payment.received` | Финансы | Двигать в «Оплачено», обнулить просрочку |

**Исходящие** (эмиты, см. `events.py` / `routes.py`):

| Событие | Получатель | Когда |
|---|---|---|
| `office.doc.created` | Дашборд владельца | Документ заведён |
| `office.shipment.requested` | Склад | Стадия «Готово к отгрузке» |
| `logistics.delivery.requested` | **Логистика** | Создана заявка перевозчику |
| `office.docs.collected` | Финансы | Стадия «Сбор документов» |
| `office.payment.awaiting` | Финансы | Стадия «Ожидают оплаты» (флаг `needs_approval` при сумме > 10 000 BYN) |
| `office.payment.overdue` | Юрист | Просрочка дебиторки > 15 дней → основание для претензии |

Шина — transactional outbox (`core/services/eventbus.py`): событие пишется в той же
транзакции, что и изменение БД (гарантия at-least-once), доставка подписчикам — relay.

## Заявка перевозчику на доставку по РБ

Кнопка **«Создать заявку перевозчику»** на карточке стадии «Готово к отгрузке»
открывает модалку с выбором перевозчика из справочника (`carriers.py`):

СДЭК · Европочта · Белпочта · Деловые Линии · DPD · Свой транспорт
(тип, рейтинг, срок, цена от, зона, флаг тяжёлого груза).

После выбора направления и даты забора создаётся номер заявки `ЛОГ-2026-NNNN`,
перевозчик фиксируется в документе, и в Логистику уходит событие
`logistics.delivery.requested`.

> AI-подбор оптимального перевозчика по весу/гео/цене — **Итерация 1** (в прототипе
> отрисован как ghost-плейсхолдер «Итерация 1 · ИИ / Недоступно в прототипе»).

## API-эндпоинты

| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/office/docs` | Документы по сделкам (список) |
| `GET` | `/office/board` | Канбан-воронка по стадиям |
| `GET` | `/office/carriers` | Справочник перевозчиков по РБ |
| `POST` | `/office/docs` | Создать документ (авто-номер `ДОК-2026-NNNN`) → `office.doc.created` |
| `PATCH` | `/office/docs/{id}` | Сменить стадию → событие соседнему отделу |
| `POST` | `/office/docs/{id}/carrier-request` | Заявка перевозчику → `logistics.delivery.requested` |

## Стадии воронки (`stages.py`, без изменений)

`ready` Готово к отгрузке · `shipped` Отгружено · `docs` Сбор документов ·
`await_pay` Ожидают оплаты · `paid` Оплачено.

## Файлы

- `module.py` — `OfficeModule`: роутер, виджет, права/роль, подписки на события.
- `events.py` — обработчики входящих событий + emit-хелперы исходящих.
- `routes.py` — эндпоинты `/office` (включая carriers и carrier-request).
- `carriers.py` — справочник перевозчиков по РБ + `get_carrier`.
- `models.py` — ORM `OfficeDoc` (+ поля связей `*_ref`, доставка, `overdue_days`).
- `schemas.py` — Pydantic-схемы (вкл. `CarrierOut/CarrierRequest/CarrierRequestOut`).
- `stages.py` — 5 стадий воронки.
- `office-manager.html` — интерактивный макет UI (превью без бэкенда).

## Миграция БД (важно)

Модель `OfficeDoc` получила новые колонки: `region`, `weight`, `address`,
`sales_ref`, `wms_ref`, `logistics_ref`, `finance_ref`, `legal_ref`, `overdue_days`.
Все добавлены **аддитивно** с `server_default`, поэтому миграция безопасна для
существующих строк. Сгенерировать и применить:

```bash
alembic revision --autogenerate -m "office: связи отделов, доставка, заявка перевозчику"
alembic upgrade head
```
